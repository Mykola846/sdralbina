# Квест на день рождения по QR-кодам — дизайн

Дата: 2026-06-14

## Суть

Сайт-квест на день рождения. Именинник сканирует QR-код, попадает на
приветственную страницу, и дальше по цепочке ищет спрятанные по дому QR-коды.
Каждый этаж дома = уровень. На каждом этаже спрятан мини-подарок и QR-код,
который показывает подсказку (фото / открытка / видео), где искать следующий.
В финале — тортик. 🎂

## Параметры

- **Механика:** линейная цепочка. Код этажа N → подсказка к коду этажа N+1.
- **Уровней:** 6 этажей.
- **Подсказки:** разного типа — фото, открытки (картинки), видео (mp4 локально
  или ссылка YouTube/Google Drive).
- **Награды:** на каждом этаже мини-подарок, в финале — тортик.
- **Стиль:** праздничный яркий — конфетти, шарики, анимации, тёплые цвета.
- **Хостинг:** GitHub Pages (бесплатный онлайн).
- **Язык:** русский.

## Поток квеста

1. **Стартовый QR** (даётся имениннику в руки / клеится на дверь) → страница
   «С днём рождения! Тебя ждёт приключение» + первая подсказка к коду 1-го этажа.
2. **QR этажа N** → «🎁 Ты нашёл мини-подарок!» + подсказка к коду этажа N+1.
3. **QR 6-го этажа** → финальная страница: тортик + поздравление.

Всего QR-кодов: 1 стартовый + 6 этажных = **7**.

## Архитектура (одностраничный статичный сайт)

```
index.html      — одна страница, рендерит экран по hash (#start, #1..#6, #final)
style.css       — праздничный яркий стиль, анимации, адаптив под телефон
app.js          — роутинг по hash, рендер этажа, прогресс, конфетти
config.js       — ВСЁ наполнение: welcome, 6 этажей, final
qr.html         — авто-генерация всех QR для печати (base URL определяется сам)
media/          — фото, открытки, видео
lib/qrcode.min.js — генератор QR локально (без интернета при печати)
```

Чистый HTML/CSS/JS, без сборки и без Node — заливается на GitHub Pages как есть.

### Роутинг

Hash-роутинг в одном `index.html`:
- `#start` (или корень `/`) — приветствие.
- `#1` … `#6` — экраны этажей.
- `#final` — тортик.

`app.js` слушает `hashchange`, читает номер, берёт данные из `config.js`,
рендерит экран.

### Модель данных (config.js)

```js
const QUEST = {
  name: "Имя именинника",
  welcome: {
    headline: "С днём рождения! 🎉",
    message: "Тебя ждёт увлекательное приключение...",
    hint: { type: "image", src: "media/floor1.jpg",
            caption: "Первый код спрятан там, где..." }
  },
  floors: [
    { floor: "1 этаж",
      gift: "Тёплые носочки 🧦",
      hint: { type: "image",  src: "media/floor2.jpg",
              caption: "Ищи там, где живут книги 📚" } },
    // ... всего 6 этажей; у 6-го hint ведёт на #final
  ],
  final: {
    headline: "Поздравляю! 🎂",
    message: "Ты прошёл весь квест!",
    media: { type: "image", src: "media/cake.jpg" }
  }
};
```

`hint.type`: `image` | `video` | `text`.
- `image` — `<img src>` (фото, открытка).
- `video` — локальный mp4 (`<video>`) или embed-ссылка (YouTube/Drive → iframe).
- `text` — просто текстовая подсказка без медиа.

### Прогресс

- В `localStorage` хранится множество найденных этажей.
- Шапка показывает «Найдено N из 6» + полоса прогресса.
- При первом открытии нового этажа — анимация конфетти.
- Жёсткой блокировки порядка нет: порядок задаётся физическим размещением кодов.

### Генерация QR (qr.html)

- Открывается на живом сайте.
- `base = location.href` без `qr.html` → корректный домен автоматически.
- Для каждого QR (старт + 6 этажей) рисует код через `lib/qrcode.min.js`
  с подписью («Старт», «1 этаж» …) — готово к печати.

## Что правит пользователь

Только `config.js` (тексты, типы и пути подсказок, подарки) и файлы в `media/`.
Остальное менять не нужно.

## Обработка ошибок / крайние случаи

- Неизвестный hash → редирект на `#start`.
- Отсутствует медиа-файл → показываем только текст подсказки (graceful).
- Нет JS / старый браузер — статичное приветствие как минимум читается.

## Деплой

1. Создать репозиторий на GitHub, залить файлы.
2. Включить GitHub Pages (branch `main`, корень).
3. Открыть `https://НИК.github.io/РЕПО/qr.html`, распечатать коды.
4. Спрятать коды по этажам, стартовый отдать имениннику.

## Вне рамок (YAGNI)

- Бэкенд, база данных, аккаунты.
- Серверная защита от «перепрыгивания» этажей.
- Мультиязычность.
